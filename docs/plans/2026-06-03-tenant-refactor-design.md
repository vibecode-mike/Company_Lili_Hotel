# 重構計畫：把系統從「綁 LINE」改成「綁組織」

> 白話版。這份文件不講技術細節，只講「每一步做完後，你在畫面上會看到什麼」。

---

## 一、為什麼要改

**現在的狀況（問題）**

系統的每一筆資料（會員、對話、標籤、行銷、FAQ…）都掛在「一個 LINE 帳號」底下。
就像一排抽屜，每個抽屜的標籤是一個 LINE 官方帳號。

- 沒有 LINE 官方帳號的客人 → 沒有抽屜 → 不能用我們的系統。
- 想賣給「只要官網彈窗客服」或「只要 FB」的客戶 → 做不到，因為系統硬性要求先有 LINE。

**改完的狀況（目標）**

在抽屜上面，多放一個更大的櫃子，叫「**組織**」（就是一個客戶 / 一個館）。
LINE、官網彈窗、FB、PMS 這些都變成櫃子裡「可有可無的工具」。

- 有 LINE 的客戶 → 組織裡放一個 LINE 工具（跟現在一樣）。
- 沒 LINE、只要官網彈窗的客戶 → 組織裡只放官網彈窗工具，照樣能用。

## 二、現在你有幾個「組織」

我查了實際資料，你目前是兩個獨立的隔離單位：

- **漢堂館**
- **雷恩館**

改完之後，它們會各自變成一個「組織」，**資料一樣分開、不會混在一起**（漢堂看不到雷恩的會員，反之亦然）。

最高權限的帳號要同時看兩館，沿用現在右上角那個「切換館別」的下拉選單 —— 只是名字從「館別 / OA」改叫「組織」。**你的操作習慣不變。**

- 最怕的事：搬資料時，漢堂的會員跑去雷恩、或兩館資料混到。
