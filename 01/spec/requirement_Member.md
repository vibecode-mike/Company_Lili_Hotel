\# 會員管理（Member Management）



\## 1. 目的

將 LINE 官方帳號會員的基本資料與互動行為（如：加入時間、訊息紀錄、活動參與）整合至後台，透過系統化工具支援行銷、客服人員，可對會員進行精準管理，統一平台管理可簡化人工操作流程，減少重複作業。



---



\## 2. 核心功能



| 功能               | 描述                                                                                                                         |

|--------------------|------------------------------------------------------------------------------------------------------------------------------|

| 1. 會員管理       | 支援 CRUD 操作，抓取會員資訊（含 LINE 會員頭像+名稱、姓名、性別、生日、聯絡方式等）、會員標籤、活動標籤、消費紀錄，並記錄建立時間與最後互動時間。 |

| 2. 搜尋與篩選     | 支援會員搜尋、篩選、排序，透過精準或模糊方式查找目標會員。                                                                     |

| 3. 標籤管理       | 可新增或修改會員標籤、廣告互動標籤，一個會員可擁有複數標籤。                                                                 |

| 4. 訊息紀錄       | 查看會員最新互動紀錄，支援人工回覆（純文字、圖片、圖文模板）、排程發送及內部註記，確保會員問題及時解決。                       |

# 3.1～3.3 會員管理

## 功能說明
提供會員快速搜尋、篩選，支援單一溝通（1:1）與批量操作（批量訊息），提升管理與執行效率。


## 核心操作

### 搜尋
- 可通過「會員（LINE名稱）」「姓名」「標籤」「手機號碼」「Email」進行模糊搜尋，點擊 Enter 後動態顯示結果於下方列表。

### 清除全部條件
- 預設篩選規則：按「最近聊天日期」以新到舊排序。
- 點擊「清除全部條件」按鈕，恢復至預設排序，並動態更新下方會員列表。

### 新增會員
- 點擊按鈕跳轉至新增會員頁面。
- 會員成功建立後，自動返回會員列表頁，並彈窗 Toast 提示「會員新增成功」。
- 系統自動紀錄該會員 ID（唯一值），且註記加入來源為「系統新增」。

\## 4. AC（验收标准）



\### 4.1 Story: S\_MM\_001 會員管理

\- 以清單形式顯示會員清單，包含字段：LINE 會員頭像＋名稱、姓名、電子信箱、手機號碼、標籤（會員＋互動）、建立時間、最後回覆日期。

\- 點擊「查看」動作，可查看會員詳細資料，字段包括：LINE 會員頭像＋名稱、姓名、性別、生日、電子信箱、手機號碼、身分證/護照號碼（唯一值）、居住地、加入來源（LINE / CRM / PMS / ERP / 系統）、建立時間、最後回覆日期、LINE UID、標籤（會員＋互動），且需註記會員 ID 為唯一值。

\- **會員資訊自動填充**：會員新增時，若來源為 LINE 官方帳號，會員的居住地、電話號碼、Email、身分證字號或護照號碼等資訊會從 LINE 註冊資料中自動帶入系統。會員資料建立後，使用者可以新增或編輯這些資訊。

\- 點擊「聊天」動作，可進入與該會員的一對一聊天訊息紀錄。

\- 若串接 CRM，可查看該會員消費紀錄（v0.2），如：消費時間、金額、房型或套餐，並可分析消費頻率。

\- 顯示會員的建立時間與最後回覆訊息的日期。

\- 內部人員可新增備註，註記該會員的消費習慣與喜好。



\### 4.2 Story: S\_MM\_002 搜尋與篩選

\- 支援搜尋字段：LINE 名稱、姓名、電子信箱、手機號碼。

\- 支援篩選條件：會員標籤、性別、建立來源（LINE官方帳號 / CRM / 後台系統）。

\- 支援排序方式：按最近回覆日期排序，可快速定位需回覆的會員。



\### 4.3 Story: S\_MM\_003 標籤管理

\- \*\*會員標籤來源（3種）\*\*：

&nbsp; 1. 外部系統串接（如：CRM / PMS）：命中基於消費條件的標籤，如：今年消費金額達指定門檻、消費房型分類（商務房、雙人房等）、訪問頻率、上次互動時間。

&nbsp; 2. 問券蒐集模組：自動生成基於會員屬性的標籤，如：性別（男性、女性）、年齡區間（18-25 歲等）、地區（台北市等）、生日月份（8月等）。

&nbsp; 3. 後台自訂：使用者手動新增的標籤（如：VIP、黑名單）。

\- \*\*互動標籤來源（2種）\*\*：

&nbsp; 1. 訊息模板設定：廣告互動標籤（如：檔期優惠、雙十、早鳥）。

&nbsp; 2. 會員互動模板：訊息圖文模板（文字按鈕確認型等）、問券模板（會員資料、滿意度調查等）。

\- 查看會員詳細資料時，可新增或刪除後台自訂標籤。

\- **標籤編輯與管理機制**：

&nbsp; 1. **標籤池共享**：所有標籤由系統統一管理，會員標籤與互動標籤分別儲存在不同的標籤池中。標籤池在同一公司的所有帳號之間共享。

&nbsp; 2. **標籤新增流程**：當使用者在標籤輸入欄位輸入關鍵字時，下拉選單會即時顯示現有標籤池中已存在的標籤，並顯示每個標籤的會員數與標籤總數。

&nbsp; 3. **新增標籤方式**：使用者可以按下 Enter 鍵或從下拉選單中選擇現有標籤來建立新標籤，會員標籤與互動標籤會分別顯示。

&nbsp; 4. **標籤確認機制**：建立標籤後需點擊「確認」按鈕，標籤才會被正式標記在會員上。若建立了標籤但未標記任何會員就離開頁面，該標籤將自動從標籤池中移除。



\### 4.4 Story: S\_MM\_004 訊息紀錄

\- 顯示一對一聊天記錄，包含 LINE 會員頭像＋名稱，並完整記錄會員與內部人員的所有聊天內容。

\- 訊息即時更新，紀錄對話開啟日期（格式：YYYY/MM/DD（week））。

\- 訊息傳送時間顯示規則：12小時制，分5個時段（與LINE一致：上午/中午/下午/晚上/凌晨），格式為 hh:mm。

\- 訊息狀態標記：僅顯示「已讀」，未讀狀態不標記。

\- 支援發送訊息類型：純文字、上傳圖片、引用由配置區設定完成的訊息模板（系統自動生成 JSON）；可排程指定日期與時間人工發送。

\- 訊息來源識別：標註「人工回覆（顯示帳號名稱）」「訊息推播」「自動回應」。

\- 聊天頁面可部分檢視會員基本資訊（姓名、性別、生日、聯絡方式等），並可編輯備註。

\- 聊天頁面可檢視會員所有標籤（會員標籤＋互動標籤），並可新增或刪除。

\- **聊天室自動回應控制**：當系統使用者點擊進入訊息輸入欄位（onfocus）時，系統會自動暫停該 LINE 會員的自動回應功能。自動回應暫停的依據為：系統使用者是否開啟該聊天室準備發送訊息。
