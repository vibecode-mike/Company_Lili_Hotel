# 釐清問題

會員搜尋功能中電子信箱模糊搜尋的具體行為與範例為何？

# 定位

Feature：member_search_filter.feature - 「支援搜尋字段：電子信箱（模糊搜尋）」（第 61-87 行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 前綴匹配 - 僅匹配 email 開頭（如 "john" 匹配 "john@example.com"） |
| B | 包含匹配 - 任意位置匹配（如 "exam" 匹配 "john@example.com"） |
| C | 域名匹配 - 支援以 @domain 搜尋（如 "@gmail.com" 列出所有 Gmail 用戶） |
| D | 混合模式 - 支援以上所有匹配方式 |
| Short | 提供其他簡短答案（<=5 字）|

註：此 Rule 目前缺少 Example，需補充測試案例。

# 影響範圍

- 會員搜尋功能準確性
- 搜尋結果效能（不同匹配模式影響查詢速度）
- 前端搜尋 UI 提示文字

# 優先級

Medium
- Medium：影響搜尋功能完整性，但非核心阻擋項

- **回答**：D - 混合模式：支援前綴、包含、域名匹配
- **更新的規格檔**：01/spec/features/member_search_filter.feature
