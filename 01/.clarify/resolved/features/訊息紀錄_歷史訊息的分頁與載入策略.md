# 釐清問題

一對一聊天訊息紀錄的歷史訊息載入策略為何？是否支援分頁或滾動載入？

# 定位

Feature：訊息紀錄 - 一對一聊天歷史訊息顯示機制

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 一次載入所有歷史訊息（適用於訊息量少的情境） |
| B | 分頁載入（如每頁 50 筆，支援翻頁） |
| C | 無限滾動（向上滾動時自動載入更早的訊息） |
| D | 預設載入最近 100 筆，提供「載入更多」按鈕 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

影響訊息紀錄查詢效能、前端載入策略、使用者體驗、以及資料庫查詢優化

# 優先級

Medium - 影響使用者體驗與系統效能，但不阻礙基本功能

---

# 解決記錄

- **回答**：C - 無限滾動（向上滾動時自動載入更早的訊息）
- **更新的規格檔**：spec/features/message_history.feature
- **變更內容**：新增 Rule「歷史訊息採用無限滾動載入策略」，包含 3 個 Example：初次載入最新 50 筆訊息、向上滾動自動載入更早訊息、已載入所有訊息時顯示提示
- **解決時間**：2025-11-14
