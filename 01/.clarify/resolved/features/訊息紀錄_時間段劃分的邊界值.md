# 釐清問題

訊息時間段（上午/中午/下午/晚上/凌晨）的邊界值為何？例如「上午」是 06:00-12:00 還是 00:00-12:00？

# 定位

Feature：spec/features/message_history.feature Rule 關於時段劃分（約第53-86行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 凌晨 00:00-06:00、上午 06:00-12:00、中午 12:00、下午 12:00-18:00、晚上 18:00-24:00 |
| B | 凌晨 00:00-05:59、上午 06:00-11:59、中午 12:00-13:59、下午 14:00-17:59、晚上 18:00-23:59 |
| C | 上午 00:00-11:59、中午 12:00-12:59、下午 13:00-17:59、晚上 18:00-23:59（無凌晨） |
| Short | 請提供邊界值（<=5字）|

# 影響範圍

影響訊息時間顯示邏輯、時段判斷規則、使用者閱讀體驗，以及時段統計分析。

Low

# 解決記錄

- **回答**：B - 凌晨 00:00-05:59、上午 06:00-11:59、中午 12:00-13:59、下午 14:00-17:59、晚上 18:00-23:59
- **更新的規格檔**：spec/features/message_history.feature
- **變更內容**：
  1. 更新所有時段規則標題，明確標註時間範圍（如「上午時段（06:00-11:59）」）
  2. 補充各時段的邊界值範例：
     - 凌晨：00:00、02:00、05:59
     - 上午：06:00、11:59
     - 中午：12:00、12:30、13:59
     - 下午：14:00、17:59
     - 晚上：18:00、23:59
  3. 12 小時制顯示範例：14:00 → 下午 02:00、18:00 → 晚上 06:00
- **設計理由**：
  - 符合台灣日常作息習慣
  - 中午時段涵蓋 2 小時午餐時段（12:00-13:59）
  - 無縫覆蓋 24 小時，無重疊、無遺漏
  - 符合飯店業務場景（區分早餐、午餐、晚餐等時段）
  - 邊界值清晰，易於實作與理解
