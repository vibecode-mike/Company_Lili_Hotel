# 釐清問題

排程發送的訊息是否支援取消？取消的時間限制為何（如發送前 5 分鐘內不可取消）？

# 定位

Feature：spec/features/message_history.feature Rule 關於排程發送（約第126-131行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 支援取消，發送時間前任何時間都可取消 |
| B | 支援取消，但發送前 5 分鐘內不可取消（避免排程任務衝突） |
| C | 不支援取消，僅能編輯排程時間或內容 |
| Short | 其他規則（<=5字）|

# 影響範圍

影響排程任務管理、取消操作 UI 設計、排程鎖定機制、使用者體驗，以及錯誤操作補救能力。

# 優先級

Medium

# 解決記錄

- **回答**：A - 支援取消，發送時間前任何時間都可取消
- **更新的規格檔**：spec/features/create_broadcast.feature
- **變更內容**：新增「支援取消排程發送，無時間限制」規則，補充「取消排程發送」「發送前任何時間都可取消」「取消後可重新編輯並再次排程」的 Example
