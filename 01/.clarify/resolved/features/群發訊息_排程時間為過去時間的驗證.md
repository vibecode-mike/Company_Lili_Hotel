# 釐清問題

當使用者設定群發訊息的排程時間為過去時間時，系統應如何處理？

# 定位

Feature：建立訊息推播 - 排程發送功能的時間驗證規則

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 前端即時驗證，不允許選擇過去時間（日期時間選擇器限制） |
| B | 前端允許但後端驗證，儲存時顯示錯誤訊息「排程時間不可為過去時間」 |
| C | 允許設定過去時間，系統自動轉為立即發送 |
| D | 允許設定過去時間，排程任務偵測到時立即觸發發送 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

影響群發訊息排程功能的驗證規則、前端日期時間選擇器設計、後端 API 驗證邏輯、以及排程任務執行邏輯

# 優先級

Medium - 影響使用者體驗與資料完整性，但可透過多層驗證防範

---

# 解決記錄

- **回答**：A - 前端即時驗證，不允許選擇過去時間（日期時間選擇器限制）
- **更新的規格檔**：spec/features/create_broadcast.feature
- **變更內容**：新增 Example「前端即時驗證排程時間不可為過去時間」，說明日期時間選擇器限制不可選擇過去日期或時間，提供最佳使用者體驗
- **解決時間**：2025-11-14
