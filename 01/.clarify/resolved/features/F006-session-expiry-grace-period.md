# F006: Session 過期緩衝期

## 問題描述
登入系統.feature 說明 Session 於「午夜 UTC+8 或 24hr 後過期」，但未說明過期前是否有預警或緩衝期。

## 相關規格
- Feature: 登入系統.feature - Session 過期邏輯
- ERM: `LoginSession` 表

## 影響範圍
- 使用者體驗
- 資料遺失風險（編輯中的內容）
- 安全性考量

## 選項
- [ ] A. 無緩衝 - 時間到立即登出
- [ ] B. 預警提示 - 過期前 5 分鐘顯示提示
- [ ] C. 延長機制 - 使用者活動時自動延長 Session
- [ ] D. 滑動過期 - 每次請求都重置過期時間（最長 24hr）

## 優先級
**Medium** - 影響使用者體驗與資料安全

---
# 解決記錄

- **回答**：A - 無緩衝：時間到立即登出（不預警、不延長）
- **更新的規格檔**：01/spec/features/登入系統.feature
- **變更內容**：強化 Session 過期規則，明確無預警、無滑動或延長，時間到即登出並提示重新登入
