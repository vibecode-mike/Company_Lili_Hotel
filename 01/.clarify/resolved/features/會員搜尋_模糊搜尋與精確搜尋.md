# 釐清問題

會員搜尋（LINE 名稱、姓名、email、手機）是使用模糊搜尋（LIKE %keyword%）還是精確搜尋（完全匹配）？

# 定位

Feature：spec/features/member_search_filter.feature Rule 關於搜尋字段（約第6-56行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 全部使用模糊搜尋，支援部分匹配 |
| B | 全部使用精確搜尋，必須完全匹配 |
| C | email 與手機精確搜尋，姓名與 LINE 名稱模糊搜尋 |
| D | 提供切換選項，讓使用者選擇模糊或精確 |
| Short | 其他邏輯（<=5字）|

# 影響範圍

影響搜尋準確性、查詢效能、使用者體驗、資料庫索引設計，以及搜尋結果數量。

# 優先級

Medium

---
# 解決記錄

- **回答**：C - email 與手機精確搜尋，姓名與 LINE 名稱模糊搜尋
- **更新的規格檔**：spec/features/member_search_filter.feature
- **變更內容**：
  1. 「LINE 名稱」搜尋規則改為模糊搜尋（LIKE %keyword%），新增部分匹配範例
  2. 「姓名」搜尋規則改為模糊搜尋（LIKE %keyword%），新增部分匹配範例
  3. 「電子信箱」搜尋規則改為精確搜尋（= 'keyword'），新增不支援部分匹配範例與提示訊息
  4. 「手機號碼」搜尋規則改為精確搜尋（= 'keyword'），新增不支援部分匹配範例與提示訊息
  5. 所有規則標題補充搜尋方式說明（模糊搜尋/精確搜尋）
