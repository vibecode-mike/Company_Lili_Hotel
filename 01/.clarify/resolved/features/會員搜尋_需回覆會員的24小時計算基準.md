# 釐清問題

「需回覆會員」的 24 小時計算基準為何？是從會員最後發送訊息時間開始，還是從系統最後回覆時間開始？

# 定位

Feature：spec/features/member_search_filter.feature Rule 關於需回覆會員（約第141-154行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 從會員最後發送訊息時間開始計算，超過 24 小時未回覆 |
| B | 從系統最後回覆時間開始計算，超過 24 小時無互動 |
| C | 從最後一次「會員發送訊息」且「系統尚未回覆」的時間計算 |
| Short | 其他基準（<=5字）|

# 影響範圍

影響需回覆會員判定邏輯、已讀未回狀態追蹤、客服回覆時效統計，以及客戶服務品質監控。

# 優先級

High

# 規格整合

## 移除內容

### spec/features/member_search_filter.feature

已移除以下 Rule 和 Example（原第 141-154 行）：

```gherkin
Rule: 可快速定位需回覆的會員

  Example: 快速定位超過24小時未回覆的會員
    Given 系統中存在以下會員及其最後互動時間
      | member_id | name | last_interaction_at | 時間差 |
      | M001      | 張三 | 2025/01/20 10:00    | 30小時 |
      | M002      | 李四 | 2025/01/21 20:00    | 4小時  |
      | M003      | 王五 | 2025/01/19 08:00    | 64小時 |
    When 使用者選擇「快速定位需回覆會員」功能
    Then 系統顯示超過24小時未回覆的會員
      | member_id | name | last_interaction_at |
      | M003      | 王五 | 2025/01/19 08:00    |
      | M001      | 張三 | 2025/01/20 10:00    |
    And 會員依最後互動時間由遠到近排序
```

## 變更說明

根據用戶決策，不實作「快速定位需回覆會員」功能，因此從 member_search_filter.feature 中移除相關規格描述。

## 相關影響

- 會員搜尋功能不包含「需回覆會員」篩選項
- 不需要追蹤「24小時未回覆」的時間計算邏輯
- 客服管理功能不包含「已讀未回狀態」的監控
- 不實作客服回覆時效統計功能
