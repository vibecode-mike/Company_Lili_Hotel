# 釐清問題

排程發送的日期與時間使用哪個時區？如何處理夏令時調整？

# 定位

Feature：spec/features/create_broadcast.feature Rule 關於排程傳送（約第108-120行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 使用台灣時區（UTC+8），不考慮夏令時（台灣無夏令時） |
| B | 使用伺服器時區，需在設定頁面明確顯示時區資訊 |
| C | 支援多時區選擇（未來國際化需求），儲存為 UTC 並轉換顯示 |
| Short | 其他設計（<=5字）|

# 影響範圍

影響排程任務執行邏輯、時間顯示格式、跨時區部署支援，以及未來國際化擴充能力。

# 優先級

Medium

# 解決記錄

- **回答**：A - 使用台灣時區（UTC+8）顯示，後端以 UTC 儲存，台灣無夏令時無需特別處理
- **更新的規格檔**：spec/features/create_broadcast.feature, spec/erm.dbml
- **變更內容**：在 create_broadcast.feature 的排程傳送規則中補充「台灣時區 UTC+8，台灣無夏令時無需特別處理」說明；在 erm.dbml 的 Message.scheduled_datetime_utc 欄位 note 中補充「固定偏移無夏令時」說明。確認系統使用 UTC 儲存、台灣時間（UTC+8）顯示的既有設計，並明確台灣無夏令時制度
