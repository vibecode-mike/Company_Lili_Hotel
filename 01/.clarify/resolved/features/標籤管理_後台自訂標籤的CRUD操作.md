# 釐清問題

後台自訂標籤是否支援完整的 CRUD 操作（建立、讀取、更新、刪除）？刪除標籤時如何處理已標記的會員？

# 定位

Feature：標籤管理 - 後台自訂標籤的管理功能

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 支援 CRUD，刪除標籤時同時刪除 MemberTag 關聯記錄 |
| B | 支援 CRUD，刪除標籤時保留 MemberTag 但標記為「已刪除」 |
| C | 支援 CRU，不支援刪除（避免資料遺失） |
| D | 支援 CRUD，刪除前需確認無會員使用才允許刪除 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

影響標籤管理介面設計、MemberTag 資料完整性、標籤刪除的級聯處理邏輯、以及資料安全性

# 優先級

High - 涉及標籤系統的核心管理功能與資料完整性

---

# 解決記錄

- **回答**：A - 支援 CRUD，刪除標籤時同時刪除 MemberTag 關聯記錄
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：更新 MemberTag 實體的 Note 說明，新增「後台自訂標籤 CRUD 操作規則」，包含：(1) 完整的 CRUD 操作定義、(2) 刪除時級聯刪除 MemberTag 記錄（ON DELETE CASCADE）、(3) 刪除影響與確認機制、(4) 權限控制（tag.delete 權限）
- **解決時間**：2025-11-14
