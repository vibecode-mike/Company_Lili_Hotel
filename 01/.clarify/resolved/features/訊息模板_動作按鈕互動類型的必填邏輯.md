# 釐清問題

動作按鈕的互動類型（開啟網址/觸發文字/觸發圖片）是「每個按鈕必填」還是「可不設定互動類型」（純裝飾按鈕）？

# 定位

Feature：spec/features/message_template.feature Rule 關於互動類型（約第295-335行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 每個按鈕必須設定互動類型，不允許無互動的按鈕 |
| B | 可不設定互動類型，按鈕純裝飾（點擊無反應） |
| C | Template02/03/04 按鈕必填互動，其他模板可選填 |
| Short | 其他規則（<=5字）|

# 影響範圍

影響表單驗證邏輯、Flex Message JSON 生成、按鈕行為設計、使用者期待，以及互動資料完整性。

# 解決記錄

- **回答**：按鈕互動類型固定為「開啟網址」，不需要選擇互動類型。訊息模板可選擇「有按鈕」或「沒有按鈕」，若有按鈕則 URL 為必填欄位
- **更新的規格檔**：spec/erm.dbml, spec/features/message_template.feature
- **變更內容**：
  - erm.dbml: 移除 action_type, action_text, action_image 欄位，保留 action_url 欄位並更新說明為「有按鈕時必填」
  - message_template.feature: 新增「按鈕互動類型固定為開啟網址」規則，更新草稿容錯與發送驗證規則，補充「無按鈕時不驗證 URL」的 Example
