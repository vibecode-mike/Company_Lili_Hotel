# 釐清問題

一般登入的密碼複雜度要求為何？最小長度、必須包含大小寫/數字/特殊符號等規則？

# 定位

Feature：spec/features/登入系統.feature Rule 關於帳號密碼（約第10-11行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 最小 8 字元，必須包含大小寫英文、數字、特殊符號 |
| B | 最小 8 字元，必須包含英文與數字，特殊符號選填 |
| C | 最小 6 字元，無其他限制（僅禁止弱密碼如 123456） |
| Short | 請提供規則（<=5字）|

# 影響範圍

影響管理員註冊流程、密碼驗證邏輯、資訊安全等級、合規性要求，以及使用者體驗。

# 優先級

Medium

---
# 解決記錄

- **回答**：採用 erm.dbml 既有定義 - 長度 8-64 字元，至少包含 3 種類型（大寫字母、小寫字母、數字、特殊符號），不允許連續字元或超過 3 次重複字元，不允許常見弱密碼，不允許與信箱相同
- **更新的規格檔**：spec/features/登入系統.feature
- **變更內容**：在登入系統功能中新增「密碼複雜度要求」規則，包含 8 個 Example 涵蓋各種驗證情境：符合要求（3種/4種類型）、長度不足、類型不足、連續字元、重複字元、常見弱密碼、與信箱相同。與 spec/erm.dbml 的 Admin.password_hash 欄位定義保持一致
