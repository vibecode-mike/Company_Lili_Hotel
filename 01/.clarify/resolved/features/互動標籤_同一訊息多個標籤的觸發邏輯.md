# 釐清問題

當一個訊息模板設定多個互動標籤（如「檔期優惠,雙十」），會員點擊時是全部標籤都觸發，還是僅觸發主標籤？

# 定位

Feature：spec/features/label_statistics.feature Example 關於多標籤觸發（約第134-141行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 全部標籤都觸發並記錄，每個標籤 trigger_count 各自累加 |
| B | 僅觸發第一個標籤（主標籤），其他標籤為輔助描述 |
| C | 由訊息模板設定決定（可選「觸發全部」或「僅主標籤」） |
| Short | 其他邏輯（<=5字）|

# 影響範圍

影響標籤觸發邏輯、統計資料準確性、標籤命名策略、資料庫寫入次數，以及互動行為分析準確性。

# 優先級

High

# 解決記錄

- **回答**：C - 由訊息模板設定決定（可選「觸發全部」或「僅主標籤」）
- **更新的規格檔**：
  - spec/erm.dbml（MessageTemplate 新增 tag_trigger_mode 欄位）
  - spec/features/interaction_tag_setup.feature（新增多標籤觸發模式的 Rule 與 Example）
  - spec/features/label_statistics.feature（更新即時貼標 Example，區分全部觸發與僅主標籤模式）
- **變更內容**：
  - MessageTemplate 新增 `tag_trigger_mode` 欄位，值域：「全部觸發」/「僅主標籤」
  - interaction_tag 欄位 note 更新，說明支援多個標籤以逗號分隔
  - 新增 Rule「訊息模板支援設定多個互動標籤，由標籤觸發模式決定記錄邏輯」，包含兩個 Example
  - 更新 label_statistics.feature 的即時貼標 Example，區分兩種觸發模式的不同行為
