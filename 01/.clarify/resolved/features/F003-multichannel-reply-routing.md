# F003: 聊天室多渠道回覆路由

## 問題描述
聊天室多渠道 feature 說明客服可切換回覆渠道，但未說明預設回覆渠道的選擇邏輯。

## 相關規格
- Feature: 聊天室多渠道 - 跨渠道紀錄整併
- 資料結構: member Table 綁定 line_uid, fb_uid, webchat_uid

## 情境範例
- 會員綁定 LINE + Facebook + Webchat
- 客服開啟聊天室準備回覆

## 影響範圍
- 客服操作體驗
- 訊息即時性
- 會員接收偏好

## 選項
- [ ] A. 最近互動 - 使用會員最後互動的渠道（目前 Feature 提到）
- [ ] B. 會員偏好 - 增加會員偏好設定欄位
- [ ] C. 渠道優先級 - 系統設定渠道優先順序 (LINE > FB > Webchat)
- [ ] D. 強制選擇 - 客服每次必須手動選擇渠道

## 優先級
**Medium** - 影響客服操作流程

---
# 解決記錄

- **回答**：A - 最近互動：預設使用會員最後互動的渠道
- **更新的規格檔**：01/spec/features/聊天室多渠道
- **變更內容**：新增預設回覆渠道規則，依 last_interaction_at 最晚者設定預設回覆渠道，仍可手動切換；若最近渠道不可用需提示切換
