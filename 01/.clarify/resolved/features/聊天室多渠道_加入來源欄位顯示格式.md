# 釐清問題

會員列表的「加入來源」欄位在多渠道綁定時應如何顯示？

# 定位

- Feature：聊天室多渠道 - 多個 Example 提到「加入來源顯示」
- Example：「加入來源顯示「LINE / Facebook / Webchat」」

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 純文字列表，以斜線分隔（如「LINE / Facebook / Webchat」）|
| B | 渠道 ICON 列表（如 LINE 圖示 + FB 圖示 + Webchat 圖示）|
| C | 僅顯示首次加入渠道（如「LINE」），hover 時顯示完整列表 |
| D | 顯示首次加入渠道 + 數字（如「LINE +2」表示還有 2 個渠道）|
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- 會員列表 UI 欄位設計
- 會員詳情頁面資訊顯示
- 客服快速識別會員渠道綁定狀態

# 優先級

Low - UI 細節，Feature 已有文字描述

---

# 解決記錄

- **回答**：Short - 「渠道圖示＋渠道來源的帳號名稱＋UID」，複數渠道全部顯示，順序依加入時間由舊到新排序（首次加入優先顯示）
- **更新的規格檔**：spec/features/聊天室多渠道.feature
- **變更內容**：新增 Rule「會員列表加入來源欄位顯示格式」，定義渠道圖示+帳號名稱+UID 的顯示格式，並依 followed_at 排序
