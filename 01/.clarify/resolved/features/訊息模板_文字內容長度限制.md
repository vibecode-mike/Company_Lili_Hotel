# 釐清問題

MessageTemplate.text_content 文字內容欄位的長度限制為何？

# 定位

Feature：建立訊息推播 - 訊息模板文字內容欄位
ERM：MessageTemplate 實體的 text_content 屬性

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | LINE API 限制（5000 字元） |
| B | 建議限制（500 字元，如 AutoResponseMessage） |
| C | 較寬鬆限制（2000 字元） |
| D | 不限制（由資料庫 TEXT 型別決定） |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

影響前端文字輸入欄位的長度驗證、後端 API 驗證規則、LINE Messaging API 相容性、以及使用者輸入體驗

# 優先級

Medium - 影響使用者輸入限制與 LINE API 相容性

---

# 解決記錄

- **回答**：C - 較寬鬆限制（2000 字元）
- **更新的規格檔**：spec/features/message_template.feature
- **變更內容**：更新 Rule「支援編輯文字內容」標題，新增長度限制說明（2000 字元），並新增 Example「文字內容超過長度限制」驗證超過長度時的錯誤處理
- **解決時間**：2025-11-14
