# F004: 輪播圖片上傳失敗處理

## 問題描述
carousel_auto_recrop.feature 說明前端使用 Canvas API 裁切圖片，但未說明裁切或上傳失敗時的處理邏輯。

## 相關規格
- Feature: carousel_auto_recrop.feature - 前端 Canvas 裁切
- 比例規則: 1:1 (900x900), 1.91:1 (1920x1005)

## 影響範圍
- 訊息模板建立流程
- 使用者體驗
- 錯誤處理

## 選項
- [ ] A. 重新選擇 - 清除並要求重新選擇圖片
- [ ] B. 保留原圖 - 使用原始未裁切圖片
- [ ] C. 自動重試 - 前端自動重試裁切 3 次
- [ ] D. 伺服器裁切 - 回退至伺服器端裁切

## 優先級
**Low** - 邊緣情況處理

---
# 解決記錄

- **回答**：A - 全有全無：任一圖片上傳失敗即阻擋建立
- **更新的規格檔**：01/spec/features/carousel_auto_recrop.feature
- **變更內容**：新增輪播多圖上傳失敗處理規則：採全有全無策略，任一圖片上傳失敗則阻擋建立、不保留部分成功，提示重試
