# F007: 標籤規則評估時機

## 問題描述
member_tag_integration.feature 說明標籤規則（如消費金額門檻），但未說明規則何時評估。

## 相關規格
- Feature: member_tag_integration.feature - 自訂條件自動貼標
- ERM: `TagRule` 表

## 情境範例
- 管理員設定「累積消費 >= 50000 元」自動貼標「高消費客戶」
- 新消費紀錄進入系統

## 影響範圍
- 標籤即時性
- 系統效能
- 規則維護

## 選項
- [ ] A. 即時評估 - 每筆消費紀錄新增時立即評估所有規則
- [ ] B. 批次評估 - 定時任務（如每小時）批次評估
- [ ] C. 觸發評估 - 查詢會員標籤時評估
- [ ] D. 混合策略 - 高優先級規則即時評估，其他批次

## 優先級
**High** - 影響標籤即時性與系統效能

---
# 解決記錄

- **回答**：A - 事件即時：每次互動/資料寫入即時計算貼標
- **更新的規格檔**：01/spec/features/member_tag_integration.feature
- **變更內容**：新增規則說明 TagRule 評估在事件處理流程中立即執行（消費同步、互動點擊皆即時貼標），不經排程或背景批次；統計欄位同步即時更新並遵守唯一性去重
