# 釐清問題

群發訊息選擇模板時，是僅能選擇「配置區已保存的模板」，還是也可以「當下臨時建立模板」？

# 定位

Feature：spec/features/create_broadcast.feature Rule 關於模板選擇（約第6-11行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 僅能選擇配置區已保存的模板，不支援臨時建立 |
| B | 可選擇已保存模板，也可在群發頁面臨時建立並使用（不保存） |
| C | 可選擇已保存模板，也可在群發頁面建立並同時保存到配置區 |
| Short | 其他設計（<=5字）|

# 影響範圍

影響群發訊息建立流程、模板管理邏輯、使用者體驗、資料一致性，以及模板重複使用效率。

# 優先級

Medium

---
# 解決記錄

- **回答**：C - 可選擇已保存模板，也可在群發頁面建立並同時保存到配置區
- **更新的規格檔**：無需更新（spec/features/create_broadcast.feature 已完整包含此設計）
- **變更內容**：確認規格已包含兩種模板來源（從模板庫選擇 / 建立新模板）、建立時可選擇是否加入模板庫（is_in_library 欄位）、複製機制（source_template_id）、模板庫顯示與篩選等完整功能
