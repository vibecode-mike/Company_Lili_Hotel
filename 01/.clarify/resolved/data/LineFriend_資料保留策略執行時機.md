# 釐清問題

LineFriend 表的資料保留策略提到「已取消追蹤保留 90 天後可選擇性刪除或歸檔」，但未定義執行時機與自動化策略。

# 定位

ERM：`spec/erm.dbml` → Table `LineFriend` → Note「已取消追蹤（is_following = false）：保留 90 天後可選擇性刪除或歸檔」

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 每日自動清除符合條件的記錄 |
| B | 每月手動歸檔 |
| C | 保留所有記錄不刪除 |
| D | 後台提供手動刪除工具 |
| Short | 其他 |

# 影響範圍

1. 排程任務或後台操作需求  
2. `member_id` 關聯資料保護  
3. 歷史資料分析與留存策略

# 優先級

Low

理由：資料刪除屬優化議題，v0 可採保守策略避免遺失資料。

---

# 解決記錄

- **回答**：選擇 C。LineFriend 記錄全部保留，不自動刪除或歸檔，以 `is_following` 狀態區分是否仍為好友。
- **更新的規格檔**：`spec/erm.dbml`
- **變更內容**：
  1. 資料保留策略改為「所有 LineFriend 記錄皆永久保留，僅透過 is_following 標記狀態」。
- **業務影響**：
  - 避免刪除造成的追蹤落差，後續可做歷史分析。
  - 無須建立額外排程或後台工具，簡化 v0 實作。
