# 釐清問題

AdminRole 關聯表中，同一管理員可被指派多少個角色？是否有數量限制？

# 定位

ERM：AdminRole 實體的數量約束規則

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 僅允許一個角色（1:1 關係，簡化權限管理） |
| B | 不限制數量（管理員可擁有多個角色，權限為聯集） |
| C | 限制 1-3 個角色（避免權限過於複雜） |
| D | 限制 1-5 個角色（平衡彈性與複雜度） |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

影響 RBAC 權限系統設計、角色指派 UI 介面、權限計算邏輯、以及系統複雜度

# 優先級

Medium - 影響權限系統的設計複雜度與使用彈性

---

# 解決記錄

- **回答**：B - 不限制數量（管理員可擁有多個角色，權限為聯集）
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：更新 AdminRole 實體的 Note 說明，新增「角色數量限制：單一管理員可指派的角色數量無上限（彈性支援複雜權限組合需求）」
- **解決時間**：2025-11-14
