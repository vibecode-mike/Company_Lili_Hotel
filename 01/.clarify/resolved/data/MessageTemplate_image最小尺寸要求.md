# 釐清問題

MessageTemplate 上傳的圖片是否有最小尺寸要求？

# 定位

ERM：MessageTemplate 實體的 image_url 屬性
Feature：建立訊息推播 - 圖片上傳與裁切功能

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 無最小尺寸限制（只要能成功裁切即可） |
| B | 正方形模式最小 900x900、橫向模式最小 1920x1000 |
| C | 統一最小 1920x1000（確保裁切後品質） |
| D | 最小 800x800（較寬鬆的限制） |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

影響圖片上傳驗證規則、前端錯誤提示訊息、圖片裁切品質保證、以及使用者上傳體驗

# 優先級

Medium - 影響圖片品質與使用者體驗，但可透過合理預設值處理

---

# 解決記錄

- **回答**：D - 最小 800x800（較寬鬆的限制）
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：更新 MessageTemplate 實體的 image_url 屬性 Note 說明，新增「圖片尺寸要求：最小 800x800 像素（確保 LINE 訊息清晰度），前端驗證圖片尺寸，後端 API 二次驗證，拒絕小於此尺寸的圖片」
- **解決時間**：2025-11-14
