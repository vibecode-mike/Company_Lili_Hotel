# 釐清問題

Member.gender 欄位的可接受值為何？是否支援「不願透露」或其他選項？

# 定位

ERM：spec/erm.dbml Member 表格 gender 欄位（約第13行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 僅支援「男」「女」兩個選項 |
| B | 支援「男」「女」「其他」三個選項 |
| C | 支援「男」「女」「不願透露」「其他」四個選項 |
| D | 允許 NULL（未填寫） + 「男」「女」選項 |
| Short | 其他值域（<=5字）|

# 影響範圍

影響問券蒐集模組的選項設計、會員篩選功能、標籤自動貼標邏輯、資料驗證規則，以及統計分析報表。

# 優先級

Medium

---
# 解決記錄

- **回答**：D - 維持現有定義：0=不透漏 / 1=男 / 2=女
- **更新的規格檔**：spec/erm.dbml, spec/features/member_tag_management.feature
- **變更內容**：
  - erm.dbml：明確 gender 欄位為必填（不允許 NULL），預設值為 0（不透漏），值域為「0=不透漏 / 1=男 / 2=女」
  - member_tag_management.feature：擴充性別比對規則，新增 3 個 Example：(1) 問券蒐集性別標籤與欄位更新、(2) 性別值域說明、(3) 預設值為不透漏
