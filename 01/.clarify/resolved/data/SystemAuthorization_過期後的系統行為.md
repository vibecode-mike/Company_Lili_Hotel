# 釐清問題

當 SystemAuthorization 過期後，除了登入會話失效外，正在執行的操作（如排程訊息發送、PMS 同步任務）應如何處理？

# 定位

ERM：SystemAuthorization 實體的授權控制範圍與過期處理機制

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 立即中止所有操作（包含排程任務、同步任務等） |
| B | 允許當前執行中的任務完成，但不再觸發新任務 |
| C | 僅阻擋新的管理員操作，背景任務不受影響 |
| D | 提供寬限期（如 7 天），寬限期內功能正常但顯示警告 |
| Short | 提供其他簡短答案（<=5 字） |

# 影響範圍

影響授權控制的範圍定義、排程任務執行邏輯、PMS 同步機制、以及系統可用性策略

# 優先級

High - 涉及授權控制的核心功能範圍與系統安全性

---

# 解決記錄

- **回答**：A - 立即中止所有操作（包含排程任務、同步任務等）
- **更新的規格檔**：spec/erm.dbml
- **變更內容**：更新 SystemAuthorization 實體的 Note 說明，新增「過期後的系統行為：採用『立即中止所有操作』策略」，包含：(1) 登入會話立即失效、(2) 排程訊息停止觸發、(3) PMS 同步任務停止、(4) 自動回應停止、(5) 背景任務停止、(6) 檢查機制定義、(7) 續約恢復機制
- **解決時間**：2025-11-14
