# 釐清問題

LineFriend 表的資料保留策略提到「已取消追蹤保留 90 天後可選擇性刪除或歸檔」，但未明確定義執行時機與自動化策略

# 定位

ERM: LineFriend 表 Note 說明
「已取消追蹤（is_following = false）：保留 90 天後可選擇性刪除或歸檔」

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 每日自動執行排程任務，刪除 90 天前取消追蹤且無會員關聯的記錄 |
| B | 每月手動執行歸檔作業，將符合條件的記錄移至歷史表 |
| C | 保留所有記錄不刪除，僅透過 is_following 欄位標記狀態（避免資料遺失） |
| D | 提供管理後台功能，允許管理員手動選擇刪除或歸檔特定記錄 |
| Short | 提供其他簡短答案(<=5 字) |

# 影響範圍

影響範圍：
1. 需建立排程任務或手動管理介面
2. 資料刪除邏輯需考慮關聯性（member_id 不為 NULL 則永久保留）
3. 資料歸檔表結構設計（若採用歸檔方案）
4. 資料庫效能與儲存空間管理

# 優先級

Low

理由：
- 非緊急需求：v0 版可暫不實作，保留所有記錄
- 資料安全：建議採用保守策略（選項 C）避免誤刪
- 可後續優化：待累積一定資料量後再評估是否需要清理
