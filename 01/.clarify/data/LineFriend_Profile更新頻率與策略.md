# 釐清問題

LineFriend 表的 LINE Profile 更新策略提到「距離上次更新超過 N 天才重新呼叫 API」，但未明確定義 N 的具體數值

# 定位

ERM: LineFriend 表 Note 說明
「後續互動：距離上次更新超過 N 天才重新呼叫 API（避免頻繁呼叫）」

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | N = 7 天（每週更新一次，平衡資料新鮮度與 API 呼叫成本） |
| B | N = 30 天（每月更新一次，降低 API 呼叫頻率） |
| C | N = 90 天（每季更新一次，最小化 API 呼叫） |
| D | N 可透過設定檔動態調整（依據 LINE API 配額與業務需求調整） |
| Short | 提供其他簡短答案(<=5 字) |

# 影響範圍

影響範圍：
1. LineFriend 表需新增 profile_updated_at 欄位記錄最後更新時間
2. LINE Webhook 處理邏輯需檢查更新時間並決定是否呼叫 Profile API
3. 系統設定檔需定義 Profile 更新間隔參數
4. API 呼叫成本與配額管理策略

# 優先級

Medium

理由：
- 成本控制：頻繁呼叫 LINE Profile API 可能產生額外成本
- 資料新鮮度：更新間隔過長會導致會員資料過時
- 建議採用可設定方式（選項 D）以保持彈性
