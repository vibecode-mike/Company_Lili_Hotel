# 釐清問題

自動回應功能的關鍵字比對邏輯缺乏邊界條件測試，特別是空白、特殊字元、超長輸入的處理策略

# 定位

Feature: 自動回應 (auto_response.feature)
Rule: 關鍵字比對邏輯：完全匹配，不區分大小寫

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 關鍵字前後空白自動trim處理（「 訂房 」與「訂房」視為相同） |
| B | 關鍵字包含多個連續空白時正規化為單一空白（「訂  房」→「訂 房」） |
| C | 關鍵字為純空白字串時拒絕儲存並顯示錯誤訊息 |
| D | 關鍵字超過最大長度（如：100 字元）時拒絕儲存 |
| E | 關鍵字包含特殊字元（如：@#$%）時允許儲存但需完全匹配（含特殊字元） |
| Short | 提供其他簡短答案(<=5 字) |

# 影響範圍

影響範圍：
1. auto_response.feature 需補充邊界條件 Examples
2. AutoResponseKeyword 表需定義驗證規則
3. 關鍵字比對邏輯需實作正規化處理
4. 前端輸入驗證與錯誤提示

# 優先級

Medium

理由：
- 使用者體驗：邊界條件處理影響關鍵字比對準確性
- 資料品質：需防止無效關鍵字寫入資料庫
- 測試完整性：邊界條件是測試必須涵蓋的範圍
