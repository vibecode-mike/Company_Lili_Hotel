# 釐清問題

當會員僅關聯單一渠道（如只有 LINE）時，客服聊天室的渠道切換下拉選單應如何顯示？

# 定位

- Feature：聊天室多渠道 - Rule: 客服可透過下拉選單切換同一 member_id 的不同渠道聊天紀錄
- Example：「客服聊天室不提供渠道切換」（新 LINE 使用者場景）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 隱藏下拉選單，僅顯示當前渠道圖示 |
| B | 顯示下拉選單但僅有一個選項（當前渠道）且不可點擊 |
| C | 顯示下拉選單，當前渠道可選，其他渠道顯示「未綁定」並不可選 |
| D | 始終顯示三個渠道選項，未綁定的顯示為灰色並提示「尚未綁定」|
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- 前端聊天室 UI 組件
- 客服操作體驗
- 與「客服聊天室不提供渠道切換」規則的一致性

# 優先級

Low - UI 細節，Feature 已有基本定義
