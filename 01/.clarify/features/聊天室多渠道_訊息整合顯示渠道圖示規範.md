# 釐清問題

客服聊天室中整合顯示多渠道訊息時，每則訊息的渠道來源標示應使用什麼視覺元素？

# 定位

- Feature：聊天室多渠道 - Rule: 客服聊天室應顯示所有整合後訊息，並標示來源
- Example：「每則訊息標示來源 (LINE / FB / Webchat)」

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 渠道 ICON（LINE 綠色圖示、FB 藍色圖示、Webchat 灰色圖示）|
| B | 訊息背景色區分（LINE 綠色背景、FB 藍色背景、Webchat 白色背景）|
| C | 文字標籤（訊息前方顯示 [LINE]、[FB]、[Webchat]）|
| D | 頭像外框顏色（LINE 綠框、FB 藍框、Webchat 灰框）|
| Short | 提供其他簡短答案（<=5 字）|

# 影響範圍

- 前端聊天室 UI 設計
- 訊息元件樣式
- 客服識別訊息來源的效率

# 優先級

Low - UI 細節，不影響核心功能邏輯
