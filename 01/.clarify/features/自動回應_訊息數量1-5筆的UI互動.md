# 釐清問題

自動回應訊息設定「至少 1 筆、最多 5 筆」的 UI 互動方式為何？是清單編輯、逐筆新增，還是表單陣列？

# 定位

Feature：spec/features/auto_response.feature Rule 關於訊息數量（約第77-96行）

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 使用「+ 新增訊息」按鈕逐筆新增，每筆可獨立編輯/刪除，限制最多 5 筆 |
| B | 使用表單陣列一次顯示 5 個訊息欄位，使用者填寫需要的數量 |
| C | 第一筆必填，後續筆以「+ 新增更多訊息」方式展開，最多 4 次 |
| Short | 其他互動方式（<=5字）|

# 影響範圍

影響前端表單設計、使用者體驗、資料驗證邏輯、預覽區顯示方式，以及操作流程複雜度。
