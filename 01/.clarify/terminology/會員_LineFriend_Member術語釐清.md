# 釐清問題

系統中存在「會員」、「LINE 好友」、「Member」、「LineFriend」等多個相關術語，需釐清定義與使用場景

# 定位

ERM: Member 表 vs LineFriend 表
Feature files: 多處混用「會員」與「LINE 好友」術語

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | 「LINE 好友」= LineFriend 表（加入 LINE OA 的所有用戶），「會員」= Member 表（填寫問卷的完整會員） |
| B | 「會員」泛指所有用戶（LINE 好友 + CRM/PMS 會員），需依據上下文判斷具體指涉 |
| C | 前端 UI 統一使用「會員」稱呼，後端程式碼明確區分 Member 與 LineFriend |
| D | 建立術語對照表（Glossary）明確定義所有術語及其英文對應 |
| Short | 提供其他簡短答案(<=5 字) |

# 影響範圍

影響範圍：
1. 所有 Feature files 需統一術語使用
2. 前端 UI 文案需統一稱呼
3. API 文件與程式碼註解需明確區分
4. 使用者手冊與訓練文件需釐清定義

# 優先級

Medium

理由：
- 溝通清晰度：術語混淆會導致需求理解錯誤
- 文件一致性：需確保所有文件使用一致的術語
- 使用者體驗：前端 UI 術語需清晰易懂
